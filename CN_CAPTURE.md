# 国服接口结构采集（安全边界）

本流程只用于分析你本人有权访问的数据结构。它不会使私有接口访问、抓包或
自动化行为获得授权。

## 不要提供的内容

- QQ / 微信密码、短信验证码或二维码
- Cookie、`Authorization`、OAuth code、ticket、JWT、签名或任何 token
- 原始 HAR、代理证书、设备指纹、QQ 号、微信 OpenID / UnionID
- 绕过证书固定、反作弊或客户端完整性校验的工具与结果

如果官方页面、WebView 或你已经合法取得的调试环境可以直接导出 HAR，请将
原始文件只保存在你自己的电脑上。采集范围应从打开“每日商店”前开始，到商店
显示完成后立即结束；不要同时打开个人资料、好友、充值或账号安全页面。

如果应用没有正常的导出能力，或只有在绕过证书固定/客户端保护后才能读取
HTTP 内容，就停止该路线，改用官方商店分享图导入。

## Android：先采集域名差异（无 root、不解密 TLS）

Android 正式版应用通常不能导出 HAR。第一阶段可使用开源的 PCAPdroid，仅查看
应用连接的 DNS/SNI 域名，不读取 HTTPS 请求体：

1. 从 PCAPdroid 官方 GitHub、F-Droid 或 Google Play 页面安装正式版本。
2. 先在掌上无畏契约中完成正常登录，再回到 PCAPdroid。
3. 在 `Status` 中将 `App Filter` 设为“掌上无畏契约”。
4. 将 `TLS decryption` 保持关闭，将 `Dump mode` 设为 `None`。
5. 做一次基线：开始采集，打开掌瓦首页并等待 10 秒，立即停止；截取
   `Connections` 列表，记为 A。
6. 做一次商店操作：重新开始采集，从首页进入每日商店，商品显示完成后立即
   停止；截取 `Connections` 列表，记为 B。
7. 只提供 A/B 中显示的应用名、SNI/DNS 域名、端口、协议和连接状态。不要提供
   IP、请求明文、原始 PCAP 或登录过程截图。

B 相比 A 新增的域名，是后续判断商店 API/静态资源主机的候选。若域名相同，
再通过连接先后、流量大小和状态做第二轮定位，不启用 TLS 解密。

## 在本机生成结构摘要

```powershell
python tools/sanitize_har.py C:\private\valorant-cn.har `
  --output C:\private\valorant-cn.sanitized.json
```

脱敏工具只保留：

- 请求方法、泛化后的域名与路径
- query/header 的名称，不保留值
- HTTP 状态码
- JSON 字段层级和数据类型，不保留任何标量值

原始 HAR 往往等同于临时登录凭证，绝对不要上传、提交到 Git 或发到聊天中。
即使是脱敏结果，也请先人工检查其中没有账号标识，再提供
`valorant-cn.sanitized.json`。

## 后续判断点

拿到脱敏结构后，只做以下兼容性判断：

1. 商店请求是否与登录、支付和社交接口分离。
2. 是否存在只读、短期、最小权限的商店会话。
3. 响应是否包含每日商品、价格、剩余时间和配件字段。
4. 商品 ID 是否能映射到现有国际服 UUID/资源缓存。
5. 是否有公开文档或明确授权允许第三方调用。

在确认第 5 点前，不实现私有鉴权复制、签名伪造或自动重放。
